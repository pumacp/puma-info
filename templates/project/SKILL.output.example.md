# SKILL: output (derived) — example

> Generic example. `output/` is created at runtime and is git-ignored, so this
> descriptor lives at the template root. Copy it to `<project>/output/SKILL.md`
> if you want the descriptor present inside a project.

- **Role:** derived output
- **Mutability:** REGENERABLE — produced by the pipeline; safe to delete and rebuild.
- **Contains:** material produced from `primary/` (with `context/` as optional
  context): converted documents, images, infographics, translations, audio, video,
  subtitles, slides. Canonical type homes: `output/docs/`, `output/video/`,
  `output/audio/`, `output/images/`, `output/slides/`, `output/subs/`.
- **Purpose / how to use:** this is the current correct output. Do **not** hand-edit
  derived files as if they were source — change `primary/` and regenerate.
- **Provenance:** generated by puma-info targets from `primary/`; each run records
  the input file path + sha256 in the AI-use log.
